Requerimientos de la Herramienta

Requerimientos Minimos

Para el correcto funcionamiento de la herramienta necesitaremos como mínimo:

Software

Para el funcionamiento a nivel de software se necesita

  • Adobe ® Acrobat Reader 9.1.0 o posterior recomendado para documentación de vista.
  • Microsoft ® Windows ® 7 , Microsoft Windows Vista (SP2 o posterior)
  • Microsoft Windows XP Pro o Home ( 32 bits SP3, 64-bit SP2)
  • Microsoft Internet Explorer versión 6.0 o posterior. La versión 7.0 es recomendada para Microsoft Windows 7 y operativo Vista.

Hardware

Para el funcionamiento a nivel de software se necesita

  • Disco duro con 1 GB de espacio libre en disco ( o más).
  • 2 GB de RAM (o más).
  • Procesador ® de doble núcleo (o más) de Intel , 3 GHz o más rápido.
  • Tarjeta gráfica debe tener 512 MB o más de tipo DDR3 dedicada o mejor memoria.

Instalación y Configuración

Proceso básico de instalación y configuración de la herramienta Arena.

  • Descarga

    Descargar el instalador de la página principal, se necesita la creación de una cuenta. Esta cuenta puede ser de modo estudiantil o persona en general. Una vez descargado se debe utilizar una herramienta de descompresión para acceder a los archivos necesarios para su posterior instalación

  • Abrir instalador

    Una vez obtenidos los archivos necesarios procedemos a ejecutar el archivo "autorun.exe". Se abre la pantalla de instalación en la que debemos escoger la configuración que necesitamos para nuestro equipo y considerando nuestro objetivo con este sistema.

  • Configurar

    Realizar la configuración básica y por defecto del programa, escoger donde se van a guardar los archivos por defecto y seleccionar la primera opción hasta finalizar la configuración. No se necesita ningun paso extra para instalar el simulador.

  • Instalar Crystal Reports Runtime Arena

    Para la generación de reportes de la simulación utilizando las herramientas propias de Arena, tenemos que instalar el paquete Crystal Reports, el instalador de este paquete viene incluido en la carpeta de la descarga y unicamente debemos ejecutar el archivo "CRRuntime_64bit_13_0_23.msi", ubicado dentro de /Redist/Crystal

  • Manuales

    El programa nos ofrece una gran cantidad de información en los manuales y en la ayuda, para esto solo debemos abrir el programa y acercarnos a la pestaña de "Arena Product Manuals". Ahí tendremos accesos a la información que podríamos estar buscando.

Funcionamiento Básico

Funcionalidades iniciales para empezar una simulación básica
Creación de un Nuevo Modelo
Creamos un nuevo proyecto desde el menú principal del programa, accediendo a "File" y aplastando "New"
Herramientas Básicas
En el panel izquierdo tenemos un abanico de opciones, en nuestro caso utilizaremos las necesarias para procesamiento discreto.
Entradas y Salidas
Utilizando las herramientas "Create" y "Dispose" podremos visualizar un recorrido de simulación.
Configuracion de la Llegada
Podemos modificar el comportamiento de llegada de nuestras entidades, en la configuración del "Create" tenemos la posibilidad de escoger varias configuraciones, como por ejemplo el tiempo de llegada, que puede ser utilizando cualquier función matemática listada o simplemente utilizando un valor constante.
Condicionales
Se pueden utilizar condicionales que van a modificar el flujo de nuestra simulación. Se pueden aplicar varias condiciones para escoger el camino que tomará la simulación.
Reportes
Utilizando la herramienta "Record" dentro de la simulación, tendremos la posibilidad de guardar lo que pase por esa linea de flujo, para darnos la opción de visualizar los resultados una vez terminada la simulación.

Planteamiento del Problema

Detalle del ejercicio a resolver utilizando esta herramienta.

Crear una simulación para el proceso de vacunación de uno de los recintos electorales designados por el gobierno. Se necesita obtener información de estos puntos de vacunación y aplicar las siguientes consideraciones.



  • Entre el 5% y el 10% de personas no podrán vacunarse
  • La persona llega al recinto y debe esperar a que haya un médico disponible
  • La vacunación dura entre 5 a 10 minutos
  • Debe esperar 20 minutos despues de la vacuna para asegurarse de que no tiene efectos negativos de la vacuna
  • La persona recibe su certificado entre 2 a 3 minutos
  • Despues de 30 días se acerca para la segunda dosis

Conclusiones

Acotaciones finales del trabajo.

El programa Arena es una herramienta muy potente para la realización de simulaciones, nos permite utilizar diversas herramientas para optimizar los procesos. Podemos utilizar muchas herramientas que simplifican y mejoran los procesos. Considerando esto cabe recalcar que el programa tiene aspectos un tanto confusos, como programadores tenemos costumbre a hacerlo por lineas y todo escrito, teniendo un software de esta manera hace que las variables y parámetros utilizados sean dificiles de recordad y que muchas de las funcionalidades sean confusas de implementar. A todo esto hay que agregar que esta herramienta es de pago y que ofrece una versión limitada de uso. Sin embargo y apesar de todo lo anterior mencionado nos permite visualizar la simulación de una manera más comprensible y organizada, permitiendonos generar reportes y observar en tiempo real que es lo que está sucediendo.

Nuevo Modelo

Podemos crear un nuevo modelo para trabajar en la simulación

En la parte superior izquierda tenemos el botón "File", presionando este botón se nos despliega una nueva ventana, en la que veremos en la primera opción el botón de "New" que nos permitirá escoger entre los modelos predefinidos que tengamos cargados. Utilizaremos el ícono de "Blank Model" para poder acceder al nuevo modelo creado vacío.

Nuevo Modelo

Herramientas para procesamiento discreto

En la parte izquierda tenemos un menú con todas las opciones de herramientas que podremos utilizar. Incluyendo la opción de Procesamiento Discreto, que nos permite obtener las herramientas que utilizaremos para esta práctica. Estas herramientas son desplegadas en la pantalla arrastrandolas y con doble click podemos acceder a sus propiedades.

Entradas y Salidas

Utilizando herramientas simples podemos empezar nuestra simulación

Con las herramientas "Create" y "Dispose" tenemos básicamente la funcionalidad de iniciar y finalizar una entidad. Podemos acceder a la configuración de "Create" para designar los tiempos de llegada entre entidades, así mismo tendremos un contador de cuantas entidades entran y en la salida tendremos la cantidad de entidades que salieron de la simulación

Configuración del "Create"

Hay como configurar diferentes parametros de la llegada

Entre los parámetros modificables del "Create" tenemos el nombre, que puede ser modificado a nuestro antojo, así mismo podremos cambiar el tipo de entidad al que va a pertenecer, el tiempo de llegada entre entidades, podremos escoger entre una gran variedad de opciones, ya sea un valor único a modo de constante, una variable predefinida o modificada anteriormente o utilizar una expresión matemática que nos permita obtener números aleatorios por ejemplo. Tambien podemos escoger si el valor de diferencia entre las llegadas es en segundos, minutos, horas y días. Por último podremos escoger cuando se va a empezar la creación de las entidades y agregar un comentario.

Condicionales

Se pueden utilizar condicionales que modifiquen el flujo de la simulación

Utilizando las herramientas que nos ofrece este programa podemos modificar el flujo de la simulación, una de estas herramientas es la condicional. Nos permite utilizar una condición que puede ser modificada desde las propiedades de la herramienta, entonces cuando la simulación llega hasta esta condición se toma una decisión y decide por que salida debe continuar. Pueden haber salidas de True y False, o de n salidas.

Generación de Reportes

Instalado correctamente el Crystal Report podremos generar reportes de nuestra simulación

Si están instaladas todas las herramientas de Arena, podremos utilizar la generación de reportes inmediatamente después de la simulación. Por defecto tendremos esta opción para mostrarnos datos que el programa encuentra relevantes, sin embargo, podemos sacar nuestros propios valores de interes utilizando la herramienta "Report" que pondremos en la linea de flujo de la simulación para que al atravesar ese estado quede guardado en una variable que podrá ser mostrado en el reporte, en la sección de Especificados por el Usuario.
